# Language And Content Requirements

These requirements are part of the project writing standard. Apply them together with the active writing guidelines, topic brief, example articles, and product-specific writing requirements.

Priority rule:

- The language style keywords are general style requirements for this workflow.
- The content specification keywords are product-context requirements for the current product and article type.
- For a different product, content rules must be adapted from that product's own writing guidelines instead of being copied mechanically.
- If any item conflicts with the active writing guidelines, follow the active writing guidelines and surface the conflict clearly.

## Application Rules

- Phase 0 must incorporate these requirements into `00_writing_checklist.md` together with the active product writing guidelines.
- Phase 2 auditor polishing must revise the first two H2 sections according to these requirements, the active writing guidelines, and the fixed Phase 0 checklist.
- Phase 3 must continue from the Phase 2 polished text and preserve its style, density, rhythm, and product-placement pattern.
- Phase 4 final audit must apply these requirements through the fixed Phase 0 checklist.
